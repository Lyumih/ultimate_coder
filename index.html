Золотые правила

  1. Всегда присутствует независимые (скрытые) элементы информации
  2. При каждой итерации уменьшать на 1 символ
  3. Всегда присутствует длина строки
  4. Всё возможно привести к чётному ничётному
  5. Любые формы данных стремятся к формату "01"(нет/да)
  6. Очивидные правила стремятся к програничным случаям

Классификация методов. Попробовать скомбинировать

  1. Вычитание целого числа из изначального
  2. Нахождение общего просто делителя числа (кратность)
  3. Преобразование числа в другое
  4. Нахождение следующего числа из предыдущего
  5. Проверка подлинности числа шифрованием SHA/MD5/ADLER32 и др.
  6. Преобразование в другой вид информации (текст, изображение, музыка и т.д.)
  7. Дополнительный смысл в числах

Примеры свойств числа. Смотреть документацию API

  • 2512 - четное число
  • 2512 - в adler32 будет 003023
  • 2512 - в sha1 будет A3214BCFD312412DFE2
  • 2512 - сумма всех чисел 2+5+1+2 = 10
  • 2512 - бинарный поиск: число < 5000 (9999/2)
  • 2512 - сумма первой + третье цифры = 3, а второй и четвертой - 7
